## Core boundary

Agent App is the composition layer for installable intelligent applications. It packages UI, workflows, storage, workers, app backend services, Runtime intent, Context needs, Knowledge bindings, Skill references, Tool/Connector requirements, Artifacts, Evidence, Policy, QC, permissions, overlays, requirement-boundary files, install-mode metadata, and App Server bridge metadata. It does not replace adjacent standards and does not run agents by itself. Compatible hosts install apps, project entries into local catalogs, inject versioned Capability SDK handles, execute through Lime Runtime, and mediate connectors, secrets, policy, and evidence.

## Current Model

- Current entry kinds: page, panel, expert-chat, command, workflow, artifact, background-task, settings.
- Layered files: app.requirements.yaml, app.boundary.yaml, app.integrations.yaml, app.operations.yaml.
- Install file: app.install.yaml for in_lime, standalone, runtime_backed, and web_host modes.
- Bridge profile: agentRuntime.bridge for Capability SDK / Host Bridge -> Desktop Host IPC -> App Server JSON-RPC -> RuntimeCore / services.
- Runtime layer: app.runtime.yaml for lime.agent task control.
- Shared host model: user state and host capabilities are shared through governed SDK handles; app storage and app backend services stay isolated.
- Local desktop storage default: host-managed per-app SQLite. PostgreSQL is for cloud, enterprise, or team-shared backends.
- Reference examples: content-factory-app and lightweight-content-ops-app.
- Host Bridge v1: `lime.agentApp.bridge` standardizes sandbox UI theme sync, host snapshots, navigation, toast, download, and capability invocation envelopes.
