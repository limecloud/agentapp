## Core boundary

Agent App is the composition layer for installable intelligent applications. It packages UI, workflows, storage, workers, Runtime intent, Context needs, Knowledge bindings, Skill references, Tool/Connector requirements, Artifacts, Evidence, Policy, QC, permissions, overlays, and v0.7 requirement-boundary files. It does not replace adjacent standards and does not run agents by itself. Hosts install apps, project entries into local catalogs, inject versioned Capability SDK handles, execute through the host Agent Runtime, and mediate connectors, secrets, policy, and evidence.

## v0.7 current model

- Current entry kinds: page, panel, expert-chat, command, workflow, artifact, background-task, settings.
- v0.7 layered files: app.requirements.yaml, app.boundary.yaml, app.integrations.yaml, app.operations.yaml.
- v0.6 runtime layer remains: app.runtime.yaml for lime.agent task control.
- Reference examples: content-factory-app and lightweight-content-ops-app.
- Host Bridge v1: `lime.agentApp.bridge` standardizes sandbox UI theme sync, host snapshots, navigation, toast, download, and capability invocation envelopes.
